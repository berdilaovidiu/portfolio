SMART ACCESS BOX

The need for smart and secured access was discovered while working on another product. It made more sense to build it as a separate product, since a flexible access control solution could be used not only for parking lots, but also for offices, sports venues, homes, or any other area that has an electric locking mechanism.

Part of the solution was a physical box that was attached to any electric lock, the other part was a mobile app that communicates with the box via low energy bluetooth. The access was granted based on flexible and shareable virtual keys.

THE LANDING PAGE

We started building the landing page even before we started talking about the mobile app. It's goal was to present the product, get people excited, gather some comments and feedback and ultimately get some pre orders.

Since the page was going to be shared in email marketing campaigns, we expected a lot of users to see it on their mobile devices, so the responsiveness way key to the success of the project. The proposal was based on the product identity concept, using full screen high quality images with the solution, simple and clean copy and social proof.

I worked closely with the visual designer on multiple iterations of the design. The images bellow only show the first version, the one that was tested with users and the one that was released. In between, there were probably close to ten versions and a lot of skype feedback sessions, since the visual designer was remote.

THE MOBILE APP

There were two types of users for the mobile app, the ones that manage the access to a secured location and the ones that request or make use of the granted permissions.

The consumers of the service had to be able to:
  • see the in range secured locations
  • make use of their permissions to open/close in range doors
  • request access for in and out of range locations
The administrators of the access points had to be able to:
  • respond to an access request
  • define permissions to an user identified by his email address

At the time we needed the visual design ready, we were in the process of looking for a new visual designer, so I had to get my hands dirty. We started with the android version, so I used material design guidelines for navigation, layout, typography and icons.

The entire process, from the first sketches, to delivering the final specs was quite fast, mostly because the functionality was straight forward and I already had some experience with material design from a previous project.

I met with the newly hired graphic designer, the technical lead and the project manager to brainstorm ideas for the design of the print materials. After that meeting I came up with proposals for both items, in terms of content and layout.

After discussing the proposals, I worked closely with the graphic designer on the design that was going to be printed. We already had some materials in the form of high quality images and part of the copy.
