Rezolve is a marketing platform that offers a mobile shopping solution along with digital payments and e-marketing products. It allows brands and over 150000 merchants to leverage location data from geo zones and beacons and to convert visual and audio media into interactive experiences on mobile devices.

Instant buy explorations

The challenge was to design a mobile e-commerce product description page with an exciting instant purchase experience. This allows buying a product without leaving the product page and going to a standard e-commerce cart. The flow is optimised for the returning customer who has already expressed delivery and payment preferences and makes it also fun for a first time user who needs to input this data.

The accordion approach

The first time user goes through three clearly separated steps in order to complete the purchase, without leaving the product page.

The returning user is only asked to select the product options, while the rest of the sections are collapsed, but still show a summary. By tapping on the collapsed steps, the user is able to update her preferences.

The overlay approach

This is a more conventional approach, with the purchasing mechanism displayed on demand, as a bottom sheet. Both of these early explorations ended up shaping the solutions released for web and mobile.

The big redesign

The Rezolve Experience Portal (RXP) is a dashboard used by merchants to create and publish mobile engagements. When I joined the company, RXP was being redesigned in order to incorporate new capabilities, some large pieces of functionality from a recently acquired company.

Existing situation

The production version had a lot of usability issues, especially on smaller devices, and an inconsistent and outdated look and feel. There was already a set of improved designs, including the new functionality, but those were only addressing some of the aesthetic issues, while introducing more complexity.

A thorough UX audit of the existing situation brought to light some severe issues:

  • Lack of a pattern for selecting an existing entity vs creating a new one. This situation was quite frequent during campaign creation and applied to entities like: locations, beacons, watermarked assets, landing pages and lead generation forms.
  • Poor new user experience. The app is based on specific mental model that was not clearly communicated. The empty states were not informative. The new user had to put in a lot of effort before seeing a benefit.
  • Critical responsiveness issues. Central flows like the engagement creation and content creation provided a poor experience on mobile.
  • Outdated aesthetics. Aesthetics and attention to details play a huge role in creating trust.
Improved solution

The redesign was meant to solve all of the above issues and more.
